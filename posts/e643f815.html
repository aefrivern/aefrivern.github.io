数据结构

Python 数据模型

__len__ 和 __getitem__ 方法：

  • __len__ 返回长度，在 len() 函数中调用
  • __getitem__ 返回指定位置的元素，在 obj[key] 中调用

初次之外，还有 __add__ 和 __mul__ 方法

__repr__ 和 __str__ 方法：

  • __repr__ 用于调试，返回一个对象的字符串表示
  • __str__ 用于显示，返回一个对象的字符串表示

自定义类型的布尔值：__bool__ 方法

Collection API

各种运算符的重载

序列阵列

分为两类：

  • 容器序列：可以存放不同类型的数据，每个存放的对象都有一个开销
  • 扁平序列：只能存放一种类型的数据，只有在头部有开销

从可变性的角度来看：

  • 可变序列
  • 不可变序列

推荐使用 list comprehension

Tuple：

  • 记录
  • 不可变 list

unpack 序列：a, b, *rest = range(5)

match-case 相当于增强版 switch-case，支持序列的模式匹配，如 case [1, 2, 3]:

高维度的索引：a[1, 2]

list.sort 原地排序，sorted 返回新的排序列表

memoryview

NumPy

其他的队列：deque、queue、heapq、multiprocessing、asyncio

字典和集合

可以使用 ** 来解包字典

使用 | 和 & 来进行集合的并和交

defaultdict：自动创建不存在的键，如 dd = defaultdict(list)

  • __missing__：在 __getitem__ 中调用，用于处理缺失的键

其他字典：

  • ChainMap：将多个映射合并为一个单独的映射
  • Counter：计数器，用于计算可哈希对象的出现次数
  • shelf：持久化字典

如果想要创建子类型，创建 UserDict 的子类

.keys()、.items()、.values() 返回的是视图，可以动态更新

frozenset：不可变集合

Unicode 文件 vs 字节

open 一个文本文件时，一定要显式指定编码，因为不同的操作系统默认编码不同

数据类构建器

collections.namedtuple：创建一个具名元组

typing.NamedTuple：创建一个具名元组的子类，例如

@dataclass：创建一个数据类，用法类似，可以接受参数 frozen=True 来创建不可变数据类

._fields：返回字段名

可以指定默认值：

__post_init__ 在初始化之后调用，用来根据默认值生成一些属性

同样可以被用在模式匹配中，如 case ClubMember(guests=[])

对象引用、可变性和垃圾回收

is vs ==

  • is 比较的是对象的标识
  • == 比较的是对象的值

deepcopy vs copy（shadow copy）：

  • deepcopy 会递归复制对象
  • copy 只会复制对象的引用

不要使用可变类型作为默认参数，因为默认参数只会在模块加载时创建一次，如果使用可变类型，会导致所有实例共享一个对象，一种解决方法是使用 None 作为默认参数，然后在函数内部创建一个新的对象

del 语句只会删除引用，不会删除对象，对象的删除由垃圾回收器负责

函数作为对象

函数作为第一类对象

支持 map、filter、reduce、apply 等函数式编程的函数，但是不常用，更常用的是 sum、any、all 等内置函数

可调用对象：

默认情况下， 函数参数既可以是位置参数，也可以是关键字参数，但是可以限制参数的类型

  • *：只接受关键字参数
  • /：只接受位置参数

operator 模块：提供了一系列的函数，用于替代一些运算符，如 operator.add 替代 +

functioin.partial：用于固定函数的部分参数，如 inc = partial(operator.add, 1)

函数中的类型注解

def func(a: int, b: int) -> int:

可以使用 union 支持多种类型，一个比较简单的写法是 str | None

泛型集合：list[str]

tuple：tuple[int, ...]

抽象基类也可以作为类型，如 Mapping、Sequence、Iterable 等

泛型可以被参数化：

可以对 TypeVar 添加约束，如 T = TypeVar('T', int, float)，或加上 bound，如 T = TypeVar('T', bound=BaseModel)

函数类型：Callable[[int, int], int]

装饰器和闭包

@ 语法糖：@decorator 等价于 func = decorator(func)

在函数被导入时，装饰器会立即执行

global：声明全局变量

nonlocal：声明外层函数的变量

变量查找规则：

  • 如果有 global 声明，从全局变量中查找
  • 如果有 nonlocal 声明，从外层函数中查找
  • 如果是一个参数或者在函数体内被赋值，从局部变量中查找
  • 如果只被引用，没有被赋值，也不是一个参数，则从内往外依次查找

lru_cache：缓存函数的结果，可以指定缓存的大小

singledispatch：根据第一个参数的类型，调用不同的函数，如

Python 式的对象

@classmethod：类方法，第一个参数是类本身

@staticmethod：静态方法，不需要传入类或者实例

接口，协议，抽象基类

有四种定义和使用接口的方法：

  • duck typing：当看到一只鸟走起来像鸭子、游泳起来像鸭子、叫起来也像鸭子，那么这只鸟就可以被称为鸭子
  • goose typing：使用 抽象基类 Abstract Base Class(ABC)，通过运行时检查 ABC 对象
  • static typing：静态类型语言的传统方法，Python 中的 typing
  • static duck typing：通过 typing.Protocol 支持

鹅类型：

如果想要定义一个 ABC 的子类，直接继承即可

另一种方法是将一个类定义为某个 ABC 的虚拟子类，这样可以被 issubclass 等识别，但没有继承 ABC 中的任何方法和属性，使用 @ABC_name.register

Protocol 默认是静态的，可以添加 @runtime_checkable 注解在运行时也可以检查

类似于 Go，最好一个 protocol 仅支持一个操作，将多个 protocol 组合以实现更多功能
