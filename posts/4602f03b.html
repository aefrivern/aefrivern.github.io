投影变换 Projection：3D -> 2D

  • 正交投影 Orthographic Projection：物体到投影面的距离与物体到投影面的垂直距离相等
  • 透视投影 Perspective Projection：物体到投影面的距离与物体到投影面的垂直距离不相等

正交投影：

  • 中心移动到原点

光栅化 Rasterization

垂直可视角度 vertical field of view 和 宽高比 aspect ratio

这两个参数和 l,r,t,bl, r, t, bl,r,t,b 的关系：

r=aspect ratio\frac{r}{} = \text{aspect ratio} r​=aspect ratio

像素 pixel：屏幕上的一个点，像素 (x,y)(x, y)(x,y) 的中心在 (x+0.5,y+0.5)(x+0.5, y+0.5)(x+0.5,y+0.5)

三角形是基本的形状，可以通过三角形的重心坐标来表示三角形内的点

光栅化：将三角形映射到像素上

一种简单的光栅化方法：点的中心在三角形内，则为 1，否则为 0

判断某个点是否在三角形内：通过叉乘，得到该点在三角形的三条边的同一侧

一种优化方案：通过三角形的三个顶点来确定一个包围盒，然后只需要判断包围盒内的点是否在三角形内，而不需要判断所有的点

如果信号变换快，而采样慢，则会出现 aliasing

  • 锯齿
  • 摩尔纹
  • 假象

抗锯齿 Anti-aliasing 的方法：做一次模糊处理，再进行采样

超采样 Super-sampling(MSAA)：将一个像素分为若干个子像素，对每个子像素进行采样，然后取平均值

一些别的方法：

  • FXAA：对图像进行边缘检测，然后对边缘进行模糊处理
  • TAA：对图像进行时间上的模糊处理

与抗锯齿思想类似的——超分辨率：DLSS

画家算法 Painter's Algorithm：先画远处的物体，再画近处的物体

问题在于需要对物体进行排序，速度较慢

Z-Buffer：对每个像素记录一个深度值，然后对每个像素进行深度测试，如果通过则更新深度值

  • 深度测试：znew<zoldz_{new} < z_{old}znew​<zold​
  • 当然，还有附带着更新当前像素处的颜色

着色 Shading

着色 shading：对一个对象应用材质的过程

Blinn-Phong 模型：分成三个部分

  • 环境光 ambient light：所有方向的光照都是一样的（亮色）
  • 漫反射光 diffuse light：光线照射到物体表面，然后反射到各个方向（反光）
  • 镜面反射光 specular light：光线照射到物体表面，然后反射到一个方向（高光）

对每个形状着色

  • flat shading：对每个三角形着色
  • goureaud shading：对每个顶点着色，然后插值
    • 顶点处法线求法：所有相邻面的法线的平均值
  • phong shading：对每个像素着色，然后插值

当模型比较复杂时，没什么太大的区别

管线 pipeline：图形的生成、变换、光栅化、着色等过程

纹理 Texture：保存物体表面的一些属性

  • 纹理坐标 texture coordinate：纹理上的坐标，通常是 (u,v)(u, v)(u,v)
  • 纹理映射 texture mapping：将纹理映射到物体表面

重心坐标 Barycentric Coordinates：用来表示三角形内的点，公式为：

P=αP0+βP1+γP2P = α P_0 + β P_1 + γ P_2 P=αP0​+βP1​+γP2​

其中 α+β+γ=1α + β + γ = 1α+β+γ=1

如果纹理太小了，就可能出现锯齿，可以使用 texture magnification 来实现放大

一种方法是双线性插值：

即先对 uuu 进行插值：

u0=lerp⁡(u00,u10,s)u_0 = \operatorname{lerp} (u_{00}, u_{10}, s) u0​=lerp(u00​,u10​,s)

u1=lerp⁡(u01,u11,s)u_1 = \operatorname{lerp} (u_{01}, u_{11}, s) u1​=lerp(u01​,u11​,s)

再对 vvv 进行插值：

v=lerp⁡(u0,u1,t)v = \operatorname{lerp} (u_0, u_1, t) v=lerp(u0​,u1​,t)

除此之外，还有三线性插值，即使用到了周围的 16 个点

但如果纹理过大了，那么就会出现摩尔纹

mipmapping 能够做快速做范围查询来解决这个问题

  • 将纹理缩小为不同的尺寸，然后存储在不同的层级中
  • 最多开销为原来的 1.33 倍

查询的层数 D 满足 D=log⁡2LD = \log_2 LD=log2​L，L=max⁡((dudx)2+(dvdx)2,(dudy)2+(dvdy)2)L = \max(\sqrt{(\frac{\mathrm{d}u}{\mathrm{d}x})^2 + (\frac{\mathrm{d}v}{\mathrm{d}x})^2}, \sqrt{(\frac{\mathrm{d}u}{\mathrm{d}y})^2 + (\frac{\mathrm{d}v}{\mathrm{d}y})^2})L=max((dxdu​)2+(dxdv​)2​,(dydu​)2+(dydv​)2​)

有的时候，得出的层数 D 可能不是整数，这时候就需要进行插值，即分别对第 D 层和第 D+1 层进行插值，然后基于连续的 D 值再进行插值

mipmap 存在的问题是远处的纹理可能会变得模糊，可以使用 各向异性过滤 anisotropic filtering 来解决这个问题

  • 将纹理压扁，这样就可以查找与坐标轴对齐的长方形区域
  • 但是仍然无法处理斜向的情况
  • 显存开销最多为原来的 3 倍，计算量不变

EWA 纹理过滤 Elliptical Weighted Average Filtering：将纹理压扁为椭圆，然后进行加权平均

纹理也可以用来表示法线，这样就可以实现 bump mapping，实现了表面的凹凸感

  • normal mapping：法线贴图，扰动后的法线 n=(−dp/du,−dp/dv,1)n = (-\mathrm{d}p / \mathrm{d}u, -\mathrm{d}p / \mathrm{d}v, 1)n=(−dp/du,−dp/dv,1)
  • displacement mapping：置换贴图，实际上移动了顶点

几何 Geometry

隐式表示法：f(x,y,z)=0f(x, y, z) = 0f(x,y,z)=0

  • 优点：容易判断某个点是否在几何体内

显式表示法：通过参数方程来表示，缺点是难以判断某个点是否在几何体内

CSG Constructive Solid Geometry：通过一些基本的几何体来构建复杂的几何体，有三种操作：

  • 并集
  • 交集
  • 差集

距离函数 Distance Function：任何一个点到几何体的最短距离（有正负），可以使用距离函数 blend 两个对象的表面

水平集 Level Set：将距离函数的值等于 0 的点构成的集合

点云 Point Cloud：一些点的集合，可以通过点云重建几何体

多边形网格 Polygon Mesh：由一些三角形或者四边形组成的网格

贝塞尔曲线的性质：

  • 端点插值
  • 端点线段相切
  • 在仿射变换后，只需要变换控制点就能变换曲线
  • 凸包性质

一般把贝赛尔曲线分成若干段，每一段都是一个贝赛尔曲线，这样比较容易控制

  • 连续性：两个相邻的贝赛尔曲线的端点和切线相等

Spline 曲线：一种平滑的曲线，有两种：

  • B-Spline：通过一些控制点来控制曲线
  • NURBS：B-Spline 的扩展，可以控制权重

贝塞尔曲面：先在一个方向上生成多条贝塞尔曲线，以这些曲线上的点为控制点，再生成一条贝塞尔曲线，最终得到一个曲面

网格细分 Subdivision：将一个网格细分为更多的网格，有两种方法：

Loop Subdivision：针对三角形网格

  • 每次将一个三角形细分为四个三角形
  • 对于新的顶点，计算公式为 v′=38(A+B)+18(C+D)v' = \frac{3}{8}(A+B) + \frac{1}{8}(C+D)v′=83​(A+B)+81​(C+D)

Catmull-Clark Subdivision：针对一般网格

  • extraordinary point：一个顶点周围的面数不等于 4
  • 每一条边取中点，每一个面取中心，然后连接
  • 在细分后，非四边形面数量会减少

网格简化 Mesh Simplification：将一个网格简化为更少的三角形

edge collapse：将一个边缩短为一个点

可以使用 二次度量误差 quadric error metric 来判断哪个边可以被缩短，即最小化新的顶点到原来相关的平面的平方和

阴影映射 Shadow Mapping：将场景从光源的视角渲染到深度缓冲区，然后再从相机的视角渲染，对每个像素进行深度测试，如果通过则说明在阴影中

这种方法的问题在于只能处理 硬阴影 hard shadow（只有点光源），与之相对的软阴影的边缘有模糊的渐变

光线追踪 Ray Tracing

尤其是在光线反射多次的情况，光栅无法有很好的效果，所以才需要光线追踪

由于光路是可逆的，所以可以视为光线从相机发出，经过多次反射后到达光源

光线投射 Ray Casting：从相机发出光线，然后判断光线与物体的交点，选择最近的交点

Whitted-style 光线追踪：光线追踪的一种方法，分为四种光线：

  • primary ray：从相机发出的光线
  • shadow ray：从交点发出的光线，判断是否在阴影中
  • secondary ray：从交点发出的光线，判断反射光

bounding volume：包围物体的体积，可以用来加速求交，即使用一个计算起来较为简单的体积来代替物体，先判断光线和体积是否相交，如果相交，再判断光线和物体是否相交

最常用的 bounding volume 是 axis-aligned bounding box (AABB)，即与坐标轴平行的包围盒，形成了三个 对面 slab，可以通过判断光线和 slab 是否相交来判断光线和 AABB 是否相交

  • 所以当且仅当 tenter<texitt_{\text{enter}} < t_{\text{exit}}tenter​<texit​，且 texit>0t_{\text{exit}} > 0texit​>0 时，光线和盒子相交

如何将空间划分为一些格子

一个方法是使用 KD 树，即每次选择一个轴，然后选择中位数作为划分点，然后递归划分

但这种方法最大的问题是一个物体可能会被划分到多个格子中，所以不能依据空间划分，而是应该依据物体划分，即 BVH Bounding Volume Hierarchy，将物体划分为一个个包围盒，然后再划分包围盒

划分节点的方案：

  • 总是选择最长的轴
  • 在中间的对象处划分节点
  • 当节点处的对象数目小于某个阈值时，停止划分

当然，类似于 KD 树，BVH 会带来空间重叠，但这并不是什么大问题

辐射度量学 Radiometry：描述光线的强度

Radiant energy：光线的能量

Radiant flux：单位时间发出的能量

Φ=dQdtΦ = \frac{\mathrm{d}Q}{\mathrm{d}t} Φ=dtdQ​

关注的三种光线度量：

  • 从光源发出的光线——radiant intensity
  • 落到物体上的光线——irradiance
  • 光线沿着射线传播——radiance

radiant intensity：单位立体角内的辐射度量

I=dΦdωI = \frac{\mathrm{d}\Phi}{\mathrm{d}ω}I=dωdΦ​

立体角 solid angle：

Ω=Ar2Ω = \frac{A}{r^2} Ω=r2A​

E=dΦdAE = \frac{\mathrm{d}\Phi}{\mathrm{d}A}E=dAdΦ​

radiance：单位面积、单位立体角上的辐射度量

L=d2ΦdAcos⁡θdωL = \frac{\mathrm{d}^2\Phi}{\mathrm{d}A \cos θ \mathrm{d}ω}L=dAcosθdωd2Φ​

即 radiance 和 irradiance 的区别在于是否考虑了方向，两者的关系为：

E=∫ΩLcos⁡θdωE = \int_{Ω} L \cos θ \mathrm{d}ω E=∫Ω​Lcosθdω

fr(ωi,ωo)=dLo(ωo)dEi(ωi)f_r(ω_i, ω_o) = \frac{\mathrm{d}L_o(ω_o)}{\mathrm{d}E_i(ω_i)} fr​(ωi​,ωo​)=dEi​(ωi​)dLo​(ωo​)​

  • ωiω_iωi​：入射光线的方向

∫f(x)dx=1N∑i=1Nf(xi)p(xi)∫ f(x) \mathrm{d}x = \frac{1}{N} \sum_{i=1}^N \frac{f(x_i)}{p(x_i)} ∫f(x)dx=N1​i=1∑N​p(xi​)f(xi​)​

由于每次反射的光线实在是太多了，所以一般选择 N=1N=1N=1，即只追踪一根光线。当然这个方法会产生噪声，因此对于每个像素都追踪多条路径，然后取平均值

什么时候停下？使用俄罗斯轮盘赌。想要得到结果 Lo，每次以 PPP 的概率返回结果 Lo/PLo / PLo/P；以 1−P1 - P1−P 的概率得到 0。最终期望就是 Lo
