深度优先搜索 Depth-first search

来源：迷宫探索该算法最早可追溯到古希腊神话中的米诺陶洛斯迷宫的走法

设计模式：dfs 处理+询问

方法：当访问顶点时

  • 标记顶点为已访问
  • 递归访问所有与相邻的未标记过的顶点

时间复杂度为度数之和（因为每个与相连的顶点都只被访问一次）

正确性证明：

  • 若被标记了，则与相连

广度优先搜索 Bread-first search

方法：重复此过程直到队列为空

  • 从队列中移除顶点

区别：dfs 用栈，bfs 用队列

bfs 计算以为源点的最短路时间复杂度（因为每个与相连的顶点都只被访问一次）

Prim 算法 Prim's algorithm

  • 从开始贪心构造树
  • 向中添加只有一端在中的最小边
  • 重复直到条边

正确性证明：Prim 算法是贪心 MST 算法的一个特殊情况

  • 假设是连接在树上和不在树上的顶点的最小边
  • cut 是连接在树上的点集
  • 没有连接边是黑色的
  • 没有连接边有更小的权重

懒惰实现： 查找只有一端在中的最小边：维护一个只有一端在中的边的优先队列

  • 键为边，优先级为边权
  • 如果边的两端都在中，不予考虑
  • 加入所有与相连的顶点

最坏情况时间复杂度，空间复杂度

优化实现：维护与相连的顶点的优先队列，优先级为最小边

  • 如果有更短的边，减少优先级

需要使用能通过索引改变键的优先队列

  • 维护并行数组pq[]、qp[]、key[]：
    • pq[i]是在堆的i位置的键的索引
    • qp[i]是索引为i的堆的位置
    • key[i]是i的优先级
  • 使用swim(pq[k])实现decreaseKey(k, key)

KMP 算法 Knuth–Morris–Pratt

思想：

  • 如果我们匹配了 5 个字符，第六个字符匹配失败
  • 我们已知了前 6 个字符
  • 不需要备份下一个指针

DFA（deterministic finite state automaton）：

  • 有限阶段（包括开始和停止）
  • 对于字母表中的每一个字符都有一个特定的转移
  • 接受转移导向停止阶段的序列

在 KMP 中，阶段 = 已经匹配的字符数（pat[]的最长前缀，txt[0...i]的一个后缀）

和暴力方法的关键不同：

  • 需要提前计算dfa[][]
  • 指针i从不减小（最多访问 N 个字符）
  • 可以使用输入流

匹配转移：如果在阶段 j，且下一个字符c == pat.charAt(j)，到 j+1 阶段

失配转移：如果在阶段 j，且下一个字符c != pat.charAt(j)，输入流的最后 j-1 个字符是pat[1...j-1]，后面跟着 c

计算dfa[c][j]：在 DFA 上模拟pat[1...j-1]，用 c 转移（如果维护阶段 X 的话，只需要常数时间）

失配转移：对于每个阶段 j，且下一个字符c != pat.charAt(j)，设置dfa[c][j] = dfa[c][X]，然后更新X = dfa[pat.charAt(j)][X]

正则表达式：描述字符串集合的方法

DFA：识别所给字符串是否在给定集合中的机器

Kleene's 定理：

  • 对于任意 DFA，存在描述相同集合的字符串的 RE
  • 对于任意 RE，存在识别相同集合的字符串 DFA

底层抽象：NFA（Nondeterministic finite automaton）

基本计划：（类似于 KMP）【使用 Kleene's 定理】

  • 从 RE 建造 NFA
  • 用 NFA 模拟文本输入

正则表达式匹配 NFA：

  • RE 被封闭在括号内
  • 每个 RE 字符一个状态
  • 红色的 ϵ 转移（改变状态，但不扫描文本）
  • 黑色的匹配转移（改变状态并扫描文本的下一个字符）
  • 如果有序列转移到接受状态，接受

非确定性：

  • 机器可以猜测合适的状态转移序列

  • 序列是机器接受文本的证明

  • 匹配成功：有某个转移序列到达接受状态

  • 匹配失败：没有转移序列能够到达接受状态

模拟要点：保证考虑了所有可能的转移序列
