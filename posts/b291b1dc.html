WSL2

~~邪教——适用于 Linux 的 Windows 子系统（WSL2）

尽管 WSL2 中的 Linux 内核是不完整的，但几乎已经足够使用了。而且其无论是启动速度还是易用性方面都比虚拟机好得多，而且 IP 地址相同

关闭：wsl --shutdown

nushell

安装

nushell 是一款现代化的 shell，支持管道、自动补全、语法高亮等，支持多平台。最引以为豪的就是类似于函数式编程的管道设计，将其发挥到极致。

修改默认 shell：向 /etc/shells 添加 nushell 的位置，然后使用 chsh -s nushell位置 修改默认 shell

添加环境变量：向 ~/.config/nushell/custom.nu 添加 path add 目录

设计哲学

大概是汲取了 bash 的管道思想和函数式编程的思想，和 bash 设计思想中的“万物皆文本”不同，nushell 的思想更多的是“万物皆结构化表格”

例如一个命令：

获取帮助：help <命令>

事实上，bash 的很多设计都违反了管道原则，例如：./program < data.in > data.out

而对于 nushell，应该使用管道写法：open data.in | ./program | save data.out -f

可以注意到，前者是以程序/控制为中心的，程序从制定地点读取数据，程序将处理后的数据写入到指定地点；而后者是以数据/信息为中心的，数据从输入文件流到程序中，经过处理后流到输出文件中

nushell 与其说是一种解释型的脚本，倒不如说是一种编译型的语言，一切都在编译期决定，例如下面这行代码是通不过的：

字符串处理

分割字符串：

  • split row 命令从一个基于分隔符的字符串创建一个列表
  • split column 命令将从一个基于分隔符的字符串中创建一个表

使用 help str 可以查看提供的 str 命令列表，包括了查找字符、大小写转换、子字符串等

列表

更新列表：

  • 增：[1, 2, 3, 4] | insert 2 10
  • 改：update
  • 最前插入：prepend，最后插入：append

迭代：

  • each 相当于 map：[1, 2] | each { |a| echo $a }
  • where 相当于 filter
  • reduce 相当于 fold：$scores | reduce --fold 1 { |elt, acc| $acc * $elt }

访问：

  • 两种索引方法：$names.1 或 $names | get 1
  • length 和 is-empty
  • in 和 not in
  • any 和 all

转换列表：flatten

表格

查询：

  • 排序：sort-by
  • 筛选列：select
  • 前 n 行：first
  • 跳过 n 行：skip

修改：

  • 上下合并：append
  • 左右合并：merge { $second }
  • 插入：insert
  • 更新：update
  • 插入或更新：upsert

自定义命令：

标志（注释附带了文档的功能）：

别名

  • 正常别名：alias ll = ls -l
  • 带管道的别名要小括号：alias lsname = (ls | get name)

作为 shell

环境变量：

  • 一个：$env.FOO = 'BAR'
  • 多个：load-env { "BOB": "FOO", "JAY": "BAR" }
  • 删除：hide
  • 上面的环境变量都会随着使用 cd 命令而消失
  • 一次性环境变量：FOO=BAR echo $env.FOO 或者 with-env { FOO: BAR } { echo $env.FOO }
  • 永久性环境变量：在 config nu 中设置

使用外部命令：^ls

检查退出码：$env.LAST_EXIT_CODE

命令行命令

Linux 自带命令

cd 切换目录

  • 绝对地址：/home/
  • 相对自制：./path

现代的代替有 zoxide

pwd 显示当前目录的绝对路径

ls 默认查看当前目录下的文件和子目录

  • / 显示根目录下的文件和子目录
  • -a 显示隐藏的文件和目录
  • -l 列出包括文件的属性和权限等
  • -R 递归地列出目录，即把子目录也列出来
  • -h 以人类可读的形式列出，即 kB 等而不是 bit 的形式

现代的代替有 exa

mkdir 创建目录

rmdir 删除 空 目录

rm 删除文件或目录

  • -rf 删除非空目录

cp 源文件 目标文件：将源文件复制为目标文件

mv 移动文件，也可以用作文件重命名

sudo 以超级用户的身份执行命令

head 查看文件前 n 行，tail 查看文件后 n 行

which 查找命令所在位置

chmod 修改文件的权限

  • +x：增加“执行”权限

ln 创建文件和目录的链接：

  • 硬链接：基于文件节点创建的链接
  • 软连接：相当于 Windows 下的“快捷方式”

echo 显示文本或变量的值

tar 打包压缩/解包解压缩

  • -c 创建包
  • -x 解包
  • -f 打包/解包为文件
  • -v 处理时显示信息

man：显示某命令的使用方法，或者使用 命令 --help

现代的替代有 tldr

cat 命令用于查看文件内容，通常通过管道与 more 和 less 一起使用，从而一页页查看数据：cat 文件名

现代的替代有 bat

find 支持查找文件，参数非常多，现代化的替代品为 fd

grep 可以用于匹配输出中的字符串

  • -A 5：显示匹配位置后面的 5 行
  • -B 3：显示匹配位置前面的 3 行
  • -C 2：显示前后各 2 行
  • -s：区分大小写
  • -i：不区分大小写
  • ^a：以 a 开头的行
  • z$：以 z 结尾的行
  • -n：显示行号
  • -r：递归搜索关键字
  • -c：匹配某关键字的次数

touch 如果文件不存在，则创建文件；如果存在，则更新文件最后修改时间

现代命令

zoxide 可以自动跳转常用的目录

  • z foo：跳转到包括 foo 的目录
  • z bar/：跳转到相对目录
  • z -：跳转到上一个目录

eza 默认彩色输出，默认配置更适用于人类，只说不同的地方

  • --icons：包括图标
  • --tree --level={{3}}：3 层的树状目录

如果浏览的是二进制文件，可以使用 hexyl

如果浏览的是 json 文件，可以使用 jq

TLDR：Too Long; Don't Read，即“太长不看”，故其优点在于简洁，使用：tldr 命令

bat：相比 cat 支持了语法高亮

  • -A：显示不可打印的字符

fd 比 find 更快，更友好

  • fd netfl：直接搜索文件名中包括 netfl 的文件
  • 支持 正则表达式
  • fd passwd /etc：第二个参数为搜索的目录
  • fd：没有参数则表示递归地列出当前目录所有文件
  • -e md 查找后缀名为 .md 的所有文件
  • -g libc.so：查找完全匹配 libc.so 名字的文件
  • -x 或 --exec：对所有搜索出来的文件实现外部程序
  • --hidden或-H --no-ignore或-I：查找隐藏文件和忽略文件

fzf 支持模糊搜索

Ncdu 用来代替 du，可以分析磁盘占用，上下键移动，Enter 键进入，d 键删除，q 键退出

htop 是 top 的替代品，以更加交互和用户友好的方式监控资源利用率

procs 是 ps 命令的替代品

  • –-tree 以树状图显示

Bash

全名为 Bourne-Again Shell，Linux 和 Unix 的默认 shell，古老且难用，但由于其历史悠久，故还有很多人在使用。

脚本通常以 shebang 开头，指定该脚本的解释器，如 #!/usr/bin/bash，即表示该脚本需要调用 /usr/bin/bash 运行

sh hello.sh：运行脚本 hello.sh；或者将该脚本权限设置为可执行的，然后直接 ./hello.sh 运行

变量是没有类型的，其赋值语句不能有空格：name='world'

使用的时候必须带上 $：echo $name

特别的，$? 保存的是上个命令的返回值，0 表示成功退出，其余情况表示出现了异常

特殊的 位置参数 可以使用传递给脚本的参数，$0 是脚本名，$1 是第一个参数，$# 是传递给脚本的参数个数

由于脚本不考虑类型，所以算术必须这么写：let x=1+1
