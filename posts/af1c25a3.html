类型系统

Go 中的指针不支持算术运算，不能被随意转换为另一个指针类型，也不能与其他指针类型进行比较

结构体字面值的写法：

两个结构体可以互相赋值，只要它们的字段类型和顺序一致，相当于复制了一份

在 Go 中，一些类型的值可能占据多个内存块，这些分布在不同内存块上的部分称为 值部，通常有一个直接值部和多个被其引用的间接值部

在赋值中，只会复制直接值部，而不会复制间接值部

数组、切片、映射

  • 数组：[N]T
  • 切片：[]T
  • 映射：map[K]T

切片相当于一个动态数组

delete(m, key) 可以删除映射中的一个键值对

数组和切片支持取子数组

可以使用 copy 函数复制切片

支持 for range 语法遍历一个容器中的键值和元素，注意被遍历的容器值是一个副本，即只有直接部分被复制了，所以尽量不要遍历大数组

clear 函数可以清空切片和映射

函数

函数签名的写法：

所有函数参数都是值传递，和赋值一样，只会复制直接值部

所有函数都可以看作是闭包

通道

通道可以是单向的，如 chan<- int 表示只能发送，<-chan int 表示只能接收，也可以是双向的，如 chan int

通道有缓冲和非缓冲之分，缓冲通道可以在初始化时指定缓冲区大小，非缓冲通道的缓冲区大小为 0

通道操作：

  • ch <- x 发送
  • x = <-ch 接收
  • x, ok = <-ch 接收，ok 表示通道是否关闭
  • close(ch) 关闭通道

创建通道：ch := make(chan int)

通道元素值的传递都是复制过程

for x := range ch 通道关闭后，循环结束

select 语句可以监听多个通道

  • 当其中一个通道有数据时，就执行对应的 case 语句
  • 如果有多个通道同时有数据，就随机选择一个执行
  • 如果没有通道有数据，就执行 default 语句

接口

接口是一种类型，但是没有具体实现，只是一组方法签名的集合

接口实现：一个类型只要实现了接口中的所有方法，就可以看作是这个接口的实现，不需要显式声明

接口可以用于实现多态

类型断言：x.(T)，如果 x 不是 T 类型，会触发 panic

type-switch：switch v := x.(type)，例如：

类型内嵌

结构体中的一个字段可以仅是一个字段类型，称为类型内嵌

类型内嵌主要是为了将被嵌入类型的方法“继承”到外部类型

实际上，内嵌类型的方法会被提升到外部类型，但是如果外部类型和内嵌类型有同名方法，外部类型的方法会覆盖内嵌类型的方法

选择器默认会选择最外层的方法，如果想要调用内嵌类型的方法，可以显式调用

恐慌和恢复

panic 会导致程序崩溃，但是可以通过 recover 来捕获恐慌，类似于异常处理

recover 只能在延迟函数中调用，如果没有在延迟函数中调用，会返回 nil

sync 标准库包中提供的并发同步技术

sync.WaitGroup 用于等待一组协程的结束

  • Add 方法用于增加计数
  • Done 方法用于减少计数
  • Wait 方法用于等待计数器归零

sync.Once 用于保证某个操作只执行一次

  • Do 方法用于执行操作

sync.Mutex 用于保护共享资源，sync.RWMutex 用于读写锁

sync.Cond 用于条件变量

  • Wait 方法用于等待条件变量
  • Signal 方法用于唤醒一个等待的协程
  • Broadcast 方法用于唤醒所有等待的协程
  • 定义条件变量时需要传入一个锁，条件变量只有在锁的保护下才能使用

测试框架

自带的测试框架

测试文件必须是 _test.go 结尾的文件，测试函数必须是 Test 开头的函数

t.Run 可以用于分组测试

运行测试：go test

  • -v 参数可以显示详细信息
  • -run 参数可以指定运行的测试函数，支持正则表达式
  • -cover 参数可以显示测试覆盖率

基准测试：Benchmark 开头的函数

  • go test -bench=. 运行基准测试
  • -benchmem 参数可以显示内存分配情况

go doc

本地网页预览工具：pkgsite，也可以直接使用 go doc 命令在命令行查看

支持的注释类型：

  • 包
  • 命令
  • 函数
  • 类型
  • 变量
  • 常量

语法类似简单的 Markdown：

  • # 标题
  • 有序列表，注意前面要多一个空格
  • 无序列表，注意前面要多两个空格
  • 代码块，前面多一个空格
  • 因此尤其要注意缩进，否则会被识别成一行

链接形式：[Text]: url

文档链接：

  • 当前包：[Name1.Name2]
  • 其他包：[pkg.Name1]
