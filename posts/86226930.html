Numpy

对于多特征，同样可以使用梯度下降

特征缩放：两个特征的变化范围差异过大，会导致成本函数的等高线呈现椭圆形，这会导致梯度下降产生振荡。所以会将各特征归一化处理，使得呈现圆形

特征缩放的方法：

  • 除以最大值
  • 中值标准化：减去均值并除以极差
  • Z-score 标准化：减去均值并除以标准差，即形成标准正态分布

对于是否收敛，可以画出迭代次数和花费函数的图像

也可以自动测试收敛：设定一个较小的 ε\varepsilonε，如果某次迭代中，J(w⃗,b)J(\vec{w}, b)J(w,b) 的减少量 ≤ε≤ \varepsilon≤ε，则可以认为收敛

对于学习率的选取，可以使用先选择一个较小的学习率，运行一段时间，无误后改用更大的学习率，最终选择合适的学习率

神经网络

尽管神经网络尝试模拟神经元，但发展到现在，已经和大脑没什么关系了，而是一套独立的理论

神经网络在以前默默无闻，近几年又突飞猛进的原因是数据的增加，传统 AI 无法有效利用这些数据，而神经网络却能够利用越来越快的硬件和越来越多的数据来提高自己的性能

神经网络类似于一级一级的逻辑回归，每个神经元都能够提取出一些特征

神经网络中的参数的上标用于表示第几层，下标表示该层的第几个神经元

多类

多类问题：在多个类中而不是两个类的分类

使用的算法是 Softmax

对于每个类的可能性的输出，有

P(y=j∣x⃗)=aj=ezj∑k=1NezkP(y = j | \vec{x}) = a_j = \frac{e^{z_j}}{\sum_{k=1}^N e^{z_k}} P(y=j∣x)=aj​=∑k=1N​ezk​ezj​​

当然，其响应的损失函数为

loss⁡(a1,…,aN,y)=−log⁡a1,y=1,…,−log⁡aN,y=N\operatorname{loss}(a_1, …, a_N, y) = -\log a_1, y = 1, …, -\log a_N, y = N loss(a1​,…,aN​,y)=−loga1​,y=1,…,−logaN​,y=N

所以，最后的输出层当然不是一个神经元，而是有多个神经元，每个神经元分别输出属于该类的可能性

注意到计算机在计算时会由于无法精确表示而有所舍入，故可以让中间结果在最后展开以获得更好的精度，最终模型预测出的不再是概率，必须使用 softmax 函数还原

当然，对于普通的逻辑回归也一样

多标签分类问题有些类似，但是更类似于多次预测，故较直接的方法是建立多个神经网络。当然，更好的方法是训练一个能够有多个输出的神经网络

评估、改进模型

将所有数据划分为训练集、交叉验证集 cross validation 和测试集

  • 训练集用于训练模型（计算参数）
  • 交叉验证集选择模型（选择超参数）
  • 测试集作为最终模型的最终正确率，代表一般性的数据

方法类似，都是通过计算最小二乘法来选择最小的

  • JtrainJ_{\text{train}}Jtrain​ 对应 bias
  • JcvJ_{\text{cv}}Jcv​ 对应方差

正则化参数 λλλ 的选择同样会对这两个值产生影响

对于偏差和方差“过高”中的“高”需要一个比较的标准，所以需要建立一个表现基准才能得知是哪一项过高了。常见的标准如下：

  • 人类水平
  • 竞争对手
  • 基于经验猜测

另外，数据集的大小也会对误差产生影响

对于有较高偏差的情况，增加数据集误差始终高于基准

但是对于有较高方差的情况，增加数据集有利于使偏差和方差向基准靠近

综上，我们可以通过增加训练集、特征工程、正则化来修正高偏差和方差的问题

神经网络的优点在于可以随意地通过建更大的网络来降低偏差，通过增加数据来降低方差，故神经网络的兴起得到了更强的 GPU 和大数据的兴起的助力

对于误差，可以将训练中出现错误的地方划分为几类，并针对性地向训练集中添加该错误类型的数据

数据增广：可以通过旋转、失真、有意义的噪声等方法改变原有的数据以生成新数据，但注意随机或无意义的噪声的没有用的

AI = Code + Data，传统方法注重模型，但现在也有以数据为中心的方法

迁移学习：别人在很大的数据集中已经训练好了一个神经网络，可以将该网络用于自己的数据集来调优，例如修改输出层以识别不同的东西

这种做法合理的原因是每层都会提取出某些特征，如边缘、角、基本形状等，绝大多数应用都会经历这些步骤，即实现了复用

当一个模型完成训练后，通常需要部署才能实现实际使用

对于一些结果会极大地倾向于某一个值的事件，单一的整体错误率并没有什么太多的用处，而是通常需要权衡精确率 precision 和召回率 recall

一种权衡方法是计算 F1 score =2PRP+R= 2 \frac{PR}{P+R}=2P+RPR​

决策树

决策树每次分裂时始终选择熵下降最快的特征的方向，每次分裂时熵下降的值叫做信息增益 information gain

一个特征可以有多种值，可以使用独热 one-hot 编码来建立正交的特征向量

对于连续的特征，同样可以通过大于/小于等于某个值来实现二分~~（排序算法复杂度的证明）~~

回归也是一样的，只不过不是使用熵来评估每个结点，而是计算该结点的方差（也即预测始终为平均值的最小二乘法误差）

一个决策树会对数据的微小变化非常敏感，可以使用大量决策树来使算法更强健，称为 tree ensemble

建立该集合的方法叫做替换取样，即每次有放回地将取样

随机森林：使用替换取样创建训练集，对新的数据集训练一棵决策树

在分裂结点时，不是使用全部的特征，而是随机使用部分特征，通常 k=nk = \sqrt{n}k=n​

XGBoost 是一个实现决策树算法的包

决策树擅长处理结构化的数据，对于非结构化数据则不擅长，速度较快

但是决策树无法从大数据和高速 GPU 中获得帮助，故相比神经网络，逐渐落伍了

k-means

算法流程：

  • 随机初始化 K 的团簇中心
  • 重复：
    • 将点分配给团簇中心
    • 将团簇中心移动到被分配的点的中心

其中，可以有多次随机的初始化，通过计算损失函数来选择最终的聚类，损失函数是各点到中心的平均距离：
