Python 包管理器——uv

uv 是一个用于管理 Python 虚拟环境的工具，特点在于速度快

换源问题，中国人永远的痛；中科大源，中国人永远的神：

在 ~/.config/uv/uv.toml（Linux）或 %APPDATA%\uv\uv.toml（Windows）中添加如下内容：

project 管理：

  • 创建新的项目：uv init <project_name>
  • 向项目中添加新的包：uv add <package_name>
  • 移除：uv remove <package_name>
  • 运行某个脚本：uv run path/to/script.py
  • 运行一条命令：uv run "command"

Python 版本管理：

  • 安装指定版本：uv install python 3.8.5
  • 查看可用的版本：uv list python
  • 将当前 project 设置为指定版本：uv python pin 3.8.5

许多 Python 包提供的是工具，uv 对此有专门的支持：

  • 在临时环境中运行：uvx <tool_name>
  • 也可以显式安装：uv tool install <tool_name>

Pip 接口兼容：

  • 创建一个新虚拟环境：uv venv
  • 使用 uv pip 代替 pip

缓存管理：

  • uv cache prune：清理不可达的缓存
  • uv cache clean：清理所有缓存

Node.js 包管理器

npm

npm (Node Package Manager) 是 Node.js 自带的包管理器，安装了 Node.js 即可使用

常用命令：

命令 效果
npm init 创建一个新的 package.json 文件
npm install 安装 package.json 文件中指定的依赖项
npm install <package name> --save 或 -S 安装指定的依赖项，并添加到 package.json 的 dependencies 中
npm install <package name> --save-dev 或 -D 安装指定的依赖项，并添加到 package.json 的 devDependencies 中
npm update 更新 package.json 文件中指定的所有包
npm update <package name> 更新指定的包
npm uninstall <package name> 卸载指定的依赖项
npm run <script name> 运行 package.json 文件中指定的脚本
npm ls 列出当前项目中已安装的所有包及其依赖关系
npm install <package name> -g 全局安装包

可以使用 nrm 切换 npm 源：npm install nrm -g

列出可用源：nrm ls

切换源：`nrm use < registry >

Windows 包管理器——scoop

Scoop 是 Windows 下的软件包管理器，通常用来安装命令行工具

软件管理：

  • 安装软件：scoop install <software>
  • 查看已安装软件：scoop list
  • 更新全部（包括 scoop）：scoop update --all
  • 清理过时的缓存和所有旧版包：scoop cleanup --all --cache
  • 清理缓存：scoop cache rm --all

bucket 是软件源，可以添加多个 bucket，以获取更多软件：

  • 添加 bucket：scoop bucket add <name> <url>，一些推荐的 bucket：
    • extras：一些常用的软件
    • nerd-fonts：字体
    • dorado：一些软件，主要是和开发相关的
  • 列出 bucket：scoop bucket list
  • 删除 bucket：scoop bucket rm <name>

软件查找：

  • 搜索软件：scoop search <keyword>
  • 查看软件信息：scoop info <software>

配置文件管理——chezmoi

chezmoi 是一款优秀的配置文件管理工具，支持多种配置文件格式，可以将配置文件存储在 Git 仓库中，方便管理

初始化 chezmoi：chezmoi init

  • 从某个 Git 仓库初始化 chezmoi：chezmoi init <repo>

文件管理：

  • 让 chezmoi 管理某个文件：chezmoi add <file>
  • 在源目录下编辑文件：chezmoi edit <file>
  • 检查 chezmoi 是否有未提交的更改：chezmoi status 和 chezmoi diff
  • 更新 dotfile：chezmoi apply --v
  • 两者结合：chezmoi edit --apply <file>

远程仓库：

  • 添加远程仓库：chezmoi cd && git remote add origin <repo>
  • 从远程仓库拉取更新到本地并应用：chezmoi update
