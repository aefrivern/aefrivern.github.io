Kotlin 是什么

Kotlin 是 JVM 家族的一员，由 Jetbrains 开发，支持与 Java 的互操作，因此能够融入 Java 生态。

Kotlin 提供了比较好的函数式编程（Java 那个是有点差的），此外还有协程，代码简洁度更好（Java 确实有些啰嗦了）

基本类型，集合，数组

Kotlin 中的类型：

  • 整数：Byte、Short、Int、Long
  • 浮点：Float、Double
  • 字符：Char
  • 布尔：Boolean

也有相应的无符号版本，如 UInt，但实际上就是普通的有符号版本，所以不是很常用

尽管只存在包装版本，但是实际上会编译为 Java 的基础类型版本

可为空的原始类型，如 Int? 才会被编译为包装版本

必须通过 .toLong() 等方法显式类型转换

Any 类型是所有类型的父类，类似于 Object；Unit 类似于 void

Nothing 用于不会正常结束的函数的返回值类型

Kotlin 的集合和 Java 类似，但区分了只读（默认）和可变（MutableXX）集合

可以使用 Array，但推荐使用集合。Array 看起来像正常的泛型类，但实际上却会编译为 Java 数组

原始类型的数组有特殊的类，如 IntArray

泛型

Kotlin 中的泛型和 Java 类似，都会在运行时被抹去

约束泛型：<T : Number>

默认情况下，泛型是可为空的，如果想要不可为空，使用 <T : Any>

inline 函数的 reified 参数类型可以在运行时使用 is 检查：

variance 标识相同基类的两个泛型类型中的哪一个，一个参数类型是另一个类型的父类或子类

A 是 A? 的子类

当一个参数只使用在返回位置时，使用 out 声明该参数为 covariant，例如只读的 List 就被声明为 covariant

相反，使用 in 声明为 contravariant

对于函数接口，有 (Animal) -> Int 是 (Cat) -> Number 的父类

既可以在声明时指明泛型类的 variance，也可以在使用时声明

如果确切类型的参数不知道或不关心，则可以使用 <*>

类型别名：typealias Combiner = (String, String) -> String

协程

协程是建立在线程上的轻量级抽象

协程是可暂停的函数

协程能够避免阻塞线程的问题，该问题会消耗、限制系统资源

相关函数：

  • runBlocking 是连接阻塞代码和可暂停函数的桥梁
  • launch 用于开启不会返回任何值的协程
  • aync 用于异步计算值

Dispatchers 决定协程运行在哪个线程或线程池：

  • Dispatchers.Default 通用
  • Dispatchers.Main 在 UI 线程上运行操作
  • Dispatchers.IO 调用阻塞 IO 的任务

大部分调度器都是多线程的

可以在 launch 时或者使用 withContext 切换调度器

结构化并发

结构化并发让你能够控制协程正在进行的工作，避免协程逃脱取消

在创建一个新的协程时，会建立协程作用域，也可以手动创建：

  • coroutineScope 用于解耦合工作：开始几个协程，等待它们得到结果，然后返回结果
  • CoroutineScope 创建和类的生命周期关联的协程，通常和 SupervisorJob 一起用

协程上下文管理单个协程是如何执行的，沿着协程的层次结构继承

协程和协程作用域的父子层次结构是通过协程上下文中的 Job 对象建立的

暂停点 是协程可以暂停，别的协程可以开始工作的地方

通过在暂停点抛出 CancellationException 实现取消

取消异常不能被忽略，而是应该要么被重新抛出，要么不在第一时间捕捉

取消是正常发生的情况，代码应该设计处理

可以使用函数 cancel 或 withTimeoutNull 来取消

使用 ensureActive 或 yield 函数配合 isActive 属性来支持取消

通常使用协程作用域来讲协程绑定到应用的生命周期上

流

流 flow 是基于协程的抽象，用于处理随着时间出现的值

两种流：hot flow 和 cold flow

cold flow 默认是懒惰的，关联到一个收集器，使用 flow 构建，使用 emit 异步提供值：

通道流是一种特殊的 cold flow，运行通过 send 函数从多个协程中发射值：

hot flow 总是激活，关联到多个收集器，这些收集器叫 订阅者 subscribers

  • 两个实例：共享流和状态流

共享流用于跨协程广播通信值：

共享流订阅者在启动后才能收到消息，可以在创建时设置重播 replay

状态流用于管理并发系统的状态，执行的是基于相等的合并，即只会在值实际改变时发送，而不是在多次赋值相同时

cold flow 通过 shareIn 和 stateIn 可以变成 hot flow

流操作

中间操作用于将流转换为别的流，对 上游的流 upstream flow 操作，返回 下游的流 downstream flow，操作是 cold，直到终结操作执行才会被调用

大部分序列的中间操作流中也有提供，流还提供了额外的中间操作：

  • 执行转换 transform
  • 管理流被执行的上下文 flowOn
  • 在流的执行的特定阶段运行代码 onStart、onCompletion

终结操作，例如 collect，运行流的代码，或者如果是热流，订阅到流上

通过从另一个流构建器收集流，发送转换后的元素来构建自己的操作
